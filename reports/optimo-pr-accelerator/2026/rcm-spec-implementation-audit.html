Critical Architectural Divergences

Spec RequirementImplementationImpact
Recursive loop: RCM Core → Project State → (loop back) → RCM Core. The spec defines continuous refinement — each pass updates Project State which feeds the next pass. Single-pass pipeline. No recursion. Intake → analyze → dashboard → stop. To re-analyze, the user starts over from /intake. HIGH — The "Recursive" in Recursive Creativity Model is not implemented. This is the most fundamental gap.
Stage Workspaces (§7): 5 sequential stages — Exploration (Explorer primary), Evaluation (Evaluator primary), Execution (Executor primary), Risk/Safety (Threat primary), Social (Social primary). Each stage runs all 5 agents but with a primary. All 5 run in a single pass. No stage concept. Explorer, Threat, Social run parallel; Evaluator and Executor sequential — but this is a dependency chain, not workspace stages. HIGH — Each spec stage has a primary agent and activities; the current implementation treats all agents as equal peers in one pass.
Cross-Agent Review (§8): Every object undergoes review by all 5 agents. Explorer: "What possibilities?" → Evaluator: "Does it make sense?" → Executor: "Can it be built?" → Threat: "What can fail?" → Social: "Will people care?" → Only then: Project State updates. Perspectives review the founder profile, not each other's outputs (except Explorer→Evaluator→Executor chain). Threat and Social review the profile directly, not the other agents' concepts. MEDIUM — The cross-review of each concept/decision by all 5 agents is not implemented.
AI Orchestration (§13): Each agent has its own AI with agent-specific responsibilities. Each perspective has its own Claude call with agent-specific system prompts. This is correctly implemented. MATCHES SPEC

How the RCM Edge Function IS Integrated (Client-Side)

// src/lib/rcm-engine.ts — the bridge between app and Edge Function

const EDGE_FUNCTION_URL =
  SUPABASE_URL ? `${SUPABASE_URL}/functions/v1/rcm-engine` : "";

export async function runRcmAnalysis(profile, onPerspective) {
  // 1. Build goal string from founder profile
  const goal = buildGoal(profile);

  // 2. Fire Edge Function call (120s timeout)
  const realPromise = fetch(EDGE_FUNCTION_URL, {
    method: "POST",
    headers: { Authorization: `Bearer ${SUPABASE_ANON_KEY}` },
    body: JSON.stringify({
      goal,                          // ← "goal" not "objective" (spec §5)
      domainPack: "venture",         // ← matches spec §16 domain packs
      depthTier: "standard",          // ← NOT in spec — OptimoPR addition
      context: `Company: ${...}`,     // ← maps to spec's "Project State context"
      profileData: profile,          // ← entire FounderProfile as JSON
    }),
    signal: controller.signal,       // ← 120s AbortController
  });

  // 3. Animate UI progress while Edge Function runs
  for (let i = 0; i < 5; i++) {
    await sleep(1500 + rand(500));
    onPerspective?.(perspectiveNames[i], i);
  }

  // 4. Await real result; fall back to mock on any failure
  const data = realPromise ? await realPromise : null;
  if (data && Array.isArray(data.perspectives)) {
    return mapResponse(data);  // Live Claude output
  }
  return runRcmMock(profile);    // Graceful degradation
}

Implementation Fidelity: ~42% of the RCM Master Specification

Fully implemented: RCM Core agents (5/5), AI Orchestration (5/5), Deliverables, Objective
Partially implemented: Project State Schema (9/15 fields partial), Domain Packs (parameter accepted but unused), Exports (5 narrative types match Venture exports)
Not implemented: Knowledge Layer integration (0/8), Specialist Engines (0/4), Recursive loop, Stage Workspaces, Branching, Scoring System, Permissions

The core reasoning architecture is correct — 5 agents with dependency-aware parallelism, agent-specific Claude calls, structured JSON output.
The knowledge integration layer is entirely absent — the most impactful gap given all libraries are installed and ready.
The recursive loop (the "R" in RCM) is not implemented — single-pass only.

Recommended Phase 2 priorities: 1) Knowledge library integration 2) Recursive refinement loop 3) Specialist engine invocation
