1 · Non-breaking mandate (hard constraint)

⛔ Nothing may break the current build

Every step below is governed by these guardrails. The live app must keep working throughout, and any new surface stays dark until validated.

#Guardrail
G1Branch/version isolation. Build on a Lovable branch/new version. telosai.optimoventures.ai production stays pinned to the current authoritative build until a new version is explicitly promoted.
G2Everything flag-gated, default OFF in prod. New surfaces behind new flags (VITE_TELOS_PSYCH_MATRIX_V2, VITE_TELOS_LOOPS_UI, VITE_TELOS_LIBRARY_LOOKUP, VITE_TELOS_MATURATION). Existing O1–O5 flags are not touched.
G3Additive schema only. New tables/columns with FORCE ROW LEVEL SECURITY; nullable/defaulted; backfill-safe. No ALTER/DROP on the existing 28 telos_* tables' columns.
G4Additive routes/components/functions. New output lives at a new flag-gated route (e.g. /app/portrait). The 18 existing routes and 31 edge functions are left intact; engine changes are new edge functions or additive params — never destructive edits.
G5Existing flows keep working. Two-path equivalence and the new pipeline are wrapped so the current assessment / RTOS / Psych Matrix / Aims flows continue unchanged.
G6Validation gate before any prod flip. Prod flag defaults stay OFF until UAT passes on the branch preview. Promotion is a separate, explicit step.
G7Instant rollback. Flags OFF = immediate revert; branch is discardable. Rollback point = commit 5e650835 + a pre-flight Supabase backup.
